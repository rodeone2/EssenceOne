The Single Model Indexer

EssenceOne takes all Views(page-parts)and Controllers(classes and functions) and places their Path $variables in a Alphanumeric list in the order in which they are requested within each Page by the Indexer(model). This elliminates the need for php- includes() and require-once() to be placed into the files. This allows you to build your applications in two ways. One have the Views Echo the Controllers functions using php short-tags or two have your controllers echo the views or both.

The Indexer Ordered List(example users profile)

  • $p1 check_user =c2/c2a1.php
  • $p2 session_user =c2/c2b1.php(both listed above header to allow redirect)
  • $p3 headers =v2/v2a1.php
  • $p4 orizontal_menu =v2/v2b1.php
  • $p5 user_profile_process=c2/c2c1.php
  • $p6 left_menu =v2/v2c1.php
  • $p7 wall_post_process =c2/c2d3.php(c2d3.php as the 3rd user file in the folder c2/)
  • $p8 profile =v2/profile.php(page name)
  • $p9 friends_list =c2/c2e1.php(set for inclusion into the right menu below)
  • $p10 right_menu =v2/v2e1.php(echoes the functions from above c2/c2e1 file)
  • $p11 footer =v2/v2g1.php(NOTICE WE CAN CHANGE page-parts TOO BY NUMERIC ORDER. v2g1,2,3,4

Many page parts will not have to repeat.

Modules Separation

Module separation is done also using Alphanumeric naming.

  • Public(static)
  • Protected(members)
  • Private(admin)
  • Add More Modules(4,5,6,7,8 and so on)
