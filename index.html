The Single Model Indexer

EssenceOne uses a single Class Module Router. .

A sample Class Module Page Router stack(example users profile)

  • a1/check_user.php
  • a1/session_user.php (both listed above header to allow redirect)
  • v1/header.php
  • v2/horizontal_menu.php
  • m2/left_user_profile.php our data model before user for auto inclusion
  • v2/left_menu.php
  • m6/wall_post_process.php blog model for auto inclusion from module blog model m6
  • v2/content.php our view for all modules that echo model data
  • m4/friends_list.php
  • v2/right_menu.php
  • v2/footer.php (NOTICE WE CAN CHANGE page-parts TOO BY NUMERIC ORDER. v2g1,2,3,4

Many page parts will not have to repeat.

This simplifies the buid process by allowing the developer to fluidly name the files and list them into the config and indexer.

Modules Separation

Module separation is done also using Alphanumeric Naming but allows page parts of other Modules to be mixed. Folders = a configs, m-models v-views c-controllers, l libs, i images, s css, j java etc.

  • Static m1,v1,c1
  • Members m2,v2,c2
  • Groups m3,v3,c3
  • Friends m4,v4,c4
  • Add More Modules(5,6,7,8 and so on)
